Product Backlog — Sweet Treats

Project: Home-based Filipino desserts business
Products: Mango Float, Strawberry Float, Berry Float, Leche Flan, Puto, Kutsinta, Cookies

Goal:
Build a simple website where customers can browse products, check stock, and reserve via Messenger. The admin can log in to manage stock, prices, and availability.

User Stories
US-01 — Browse Products

Priority: High
Story Points: 3

Description:
As a customer, I want to see all available desserts with photos, names, and descriptions so that I can decide what to order.

Acceptance Criteria:

Homepage displays all seven products
Each product shows image, name, description, and price
Layout works on mobile and desktop
US-02 — View Stock Availability

Priority: High
Story Points: 2

Description:
As a customer, I want to see how many units are available so I know if I can reserve.

Acceptance Criteria:

Each product shows stock count (e.g., “5 left”)
“Sold Out” badge appears when stock is 0
Sold-out items are visually distinct
US-03 — Reserve via Messenger

Priority: High
Story Points: 2

Description:
As a customer, I want a button to message the seller about a product.

Acceptance Criteria:

Button opens Messenger with pre-filled product name
Button is disabled when sold out
Works on mobile and desktop
US-04 — Admin Login

Priority: High
Story Points: 3

Description:
As the admin, I want to log in securely to manage products.

Acceptance Criteria:

Login form includes username and password
Invalid login shows error
Valid login redirects to dashboard
Session persists until logout
US-05 — Update Stock

Priority: High
Story Points: 3

Description:
As the admin, I want to update stock quantities.

Acceptance Criteria:

Dashboard lists all products with stock
Stock can be edited via input or buttons
Changes reflect immediately on the site
Stock cannot go below 0
US-06 — Admin Logout

Priority: Medium
Story Points: 1

Description:
As the admin, I want to log out securely.

Acceptance Criteria:

Logout button is visible
Logout ends session and redirects to login
US-07 — Mark Product Unavailable

Priority: Medium
Story Points: 2

Description:
As the admin, I want to hide products not available for the day.

Acceptance Criteria:

Toggle availability per product
Unavailable items are hidden or labeled
US-08 — Edit Price

Priority: Medium
Story Points: 2

Description:
As the admin, I want to update product prices.

Acceptance Criteria:

Price field is editable
Price must be a positive number
Changes reflect immediately
US-09 — Product Descriptions

Priority: Medium
Story Points: 1

Description:
As a customer, I want short product descriptions.

Acceptance Criteria:

Each product has a 1–2 sentence description
Admin can edit descriptions
US-10 — Mobile-Friendly Design

Priority: High
Story Points: 2

Description:
As a customer, I want the site to work well on my phone.

Acceptance Criteria:

No horizontal scrolling
Buttons are at least 44×44 px
Images load quickly and scale properly
US-11 — Low Stock Warning

Priority: Low
Story Points: 2

Description:
As the admin, I want alerts for low stock.

Acceptance Criteria:

Stock ≤ 3 shows “Low Stock” badge
Stock = 0 shows “Sold Out” badge
US-12 — Contact and Business Hours

Priority: Medium
Story Points: 1

Description:
As a customer, I want to see contact details and operating hours.

Acceptance Criteria:

Contact section shows Messenger, phone number, and business hours
